Java线程创建的几种方式、线程池使用及状态详解

发表于2026-05-31|更新于2026-06-01|Java
|总字数:1.1k|阅读时长:4分钟|浏览量:

1. Java中线程创建的几种方式

Java中创建线程主要有四种方式：

方式一：继承 Thread 类

方式二：实现 Runnable 接口

方式三：实现 Callable 接口 + FutureTask

相比于 Runnable，Callable 可以有返回值且能抛出异常。

方式四：使用线程池（ExecutorService）

生产环境推荐方式，避免频繁创建/销毁线程的开销。

总结对比

方式 特点
继承 Thread 简单直接，但 Java 单继承，扩展性差
实现 Runnable 无返回值，但与 Thread 解耦，推荐
Callable + FutureTask 有返回值、可抛异常
线程池 复用线程、管理资源，生产环境首选

核心原理

无论哪种方式，最终都是调用 Thread.start() → native 方法 start0() → JVM 底层创建系统线程 → 回调 run() 方法。所以本质上，所有方式都是对 Thread.start() 的封装。

2. 为什么不推荐使用 Executors 工厂类创建线程池？

阿里巴巴Java开发手册明确禁止使用 Executors 创建线程池，原因是可能导致 OOM。

具体风险

Executors 方法 底层问题 风险
newFixedThreadPool(n) 使用无界 LinkedBlockingQueue 任务堆积，队列无限增长 → OOM
newSingleThreadExecutor() 同上 同上 → OOM
newCachedThreadPool() 最大线程数 = Integer.MAX_VALUE 无限创建线程 → OOM
newScheduledThreadPool(n) 核心线程数可设，但最大线程数 = Integer.MAX_VALUE 线程数膨胀 → OOM

正确做法：手动 new ThreadPoolExecutor

线程池7大参数

参数 说明
corePoolSize 核心线程数，即使空闲也保留
maximumPoolSize 最大线程数
keepAliveTime 非核心线程的空闲存活时间
unit keepAliveTime 的时间单位
workQueue 阻塞队列，必须使用有界队列
threadFactory 线程工厂，用于自定义线程名
handler 拒绝策略（Abort / CallerRuns / Discard / DiscardOldest）

四种拒绝策略

策略 行为
AbortPolicy（默认） 抛 RejectedExecutionException 异常
CallerRunsPolicy 由调用者线程直接执行该任务
DiscardPolicy 直接丢弃，不抛异常
DiscardOldestPolicy 丢弃队列最老的任务，重新提交当前任务

3. 线程池有哪几种状态？每种状态表示什么？

线程池（ThreadPoolExecutor）共有 5 种状态，源码中用 AtomicInteger 的高 3 位存储状态，低 29 位存储工作线程数。

五种状态

状态 值 描述
RUNNING 111 接受新任务 + 处理队列中的任务
SHUTDOWN 000 不接受新任务，但继续处理队列中已有的任务
STOP 001 不接受新任务，不处理队列任务，中断正在执行的任务
TIDYING 010 所有任务已终止，workerCount = 0，即将调用 terminated() 钩子
TERMINATED 011 terminated() 方法执行完毕，线程池彻底终止

状态流转

状态对应的方法

调用方法 状态变化
shutdown() RUNNING → SHUTDOWN（平滑关闭）
shutdownNow() RUNNING/STOP → STOP（暴力关闭）
自动过渡 任务执行完毕且线程数为 0 时自动进入 TIDYING → TERMINATED
