黑马点评 · 第一阶段：用户认证体系

项目：黑马点评（hm-dianping），对标大众点评核心功能的位置服务平台

当前完成：用户手机验证码登录、Token 认证拦截、用户信息查询

其他模块：Controller 骨架已建，业务未实现，前后端未联调

一、项目概述

基于 Spring Boot + MyBatis-Plus + Redis 构建的位置服务平台。当前仅完成 用户认证体系 的搭建与前后端联调，其余模块为空白骨架。

二、技术栈

技术 版本 用途
Spring Boot 2.3.12 应用框架
Java 8 运行环境
MyBatis-Plus 3.4.3 ORM 框架
MySQL 5.x 关系数据库
Redis 3.2+ 验证码存储 / Token 会话
Nginx 1.18.0 前端静态资源服务器 + API 反向代理
Vue.js 2 - 前端页面
Axios - HTTP 请求（请求拦截器自动携带 Token）
Hutool 5.7.17 Bean 复制、随机数、文件操作
Lombok - @Data、@Slf4j 等

三、项目结构

四、已完成功能（用户认证体系）

4.1 发送验证码

项目 说明
接口 POST /user/code?phone={phone}
请求方 login.html
逻辑 校验手机号格式 → 生成 6 位随机码 → 存入 Redis（key=login:code:{phone}, TTL=2min）→ 控制台打印（无真实 SMS）→ 返回成功

4.2 登录

项目 说明
接口 POST /user/login
请求体 {"phone": "xxx", "code": "xxx"}
逻辑 校验手机号 → 从 Redis 取验证码比对 → 查 MySQL（tb_user）→ 不存在则自动注册（随机昵称 user_xxx）→ 生成 UUID Token → User 转 UserDTO（id, nickName, icon）→ 存入 Redis Hash（key=login:token:{token}, TTL=36000min）→ 返回 Token 字符串

4.3 Token 验证拦截

项目 说明
拦截器 LoginInterceptor（HandlerInterceptor）
拦截范围 所有路径 /**，排除 /user/code、/user/login、/blog/hot、/shop-type/**、/shop/**、/voucher/**、/upload/**
逻辑 取 Authorization header → 查 Redis Hash login:token:{token} → 空则 401 → BeanUtil.fillBeanWithMap 转 UserDTO → 存 UserHolder（ThreadLocal）→ 刷新 TTL

4.4 获取当前用户

项目 说明
接口 GET /user/me
数据来源 UserHolder.getUser()（由拦截器提前注入）
响应 {"success": true, "data": {"id": 1, "nickName": "user_xxx", "icon": "..."}}

4.5 获取用户详情

项目 说明
接口 GET /user/info/{id}
数据来源 tb_user_info 表
响应 用户扩展信息（城市、简介、粉丝数、等级等）

五、认证流程完整时序

六、关键技术知识点

6.1 Token 认证替代 Session

对比维度 Session 方式 本项目 Token 方式
存储位置 服务端内存 Redis（独立于应用）
标识传递 Cookie（浏览器自动带） Header（前端手动带）
扩展性 粘性 Session 或共享 天然支持水平扩展
失效控制 服务端控制 Redis TTL + 刷新机制

6.2 ThreadLocal 用户上下文

每个请求独立线程，Interceptor 注入 → Controller 使用 → 请求结束自动回收。避免了在 Controller/Service 方法参数中层层传递用户对象。

6.3 Redis 数据结构实战

数据类型 用途 Key 示例 TTL
String（value） 验证码 login:code:138xxxx 2 min
Hash 用户会话 login:token:{uuid} 36000 min（~25天）

Hash 结构优势：存储结构化用户信息（id, nickName, icon），BeanUtil 系列方法可自动转换 DTO ↔ Map。

6.4 统一响应体

配合全局异常处理 WebExceptionAdvice，所有异常统一返回 {"success": false, "errorMsg": "服务器异常"}。

6.5 拦截器机制

MvcConfig 注册 LoginInterceptor：

  • preHandle(): 请求到达 Controller 前处理（认证、日志、权限）
  • 放行路径：登录、注册、公开资源等无需 Token 的接口

6.6 MyBatis-Plus 基础

  • 继承 ServiceImpl<Mapper, Entity> 获得基础 CRUD
  • MyBatis-Plus 自动驼峰映射、分页拦截器
  • @TableId 主键策略（AUTO 自增 / INPUT 手动指定）
  • @TableField(exist = false) 非持久化字段
  • LambdaQueryWrapper 条件查询

6.7 Hutool 工具链

工具方法 用途
RandomUtil.randomString(6) 生成 6 位随机验证码
BeanUtil.copyProperties(source, targetClass) 对象属性复制
BeanUtil.beanToMap(obj, map, options) 对象转 Map（字段值转 String）
BeanUtil.fillBeanWithMap(map, bean, isToCamelCase) Map 填充回对象

十、后续计划

  • 商户模块：Redis 缓存（解决穿透/雪崩）、前后端联调
  • 秒杀功能：分布式锁、Redis 预减库存、异步下单
  • 博客模块：Redis Set 点赞去重、取消点赞
  • 关注与 Feed 推送
  • 附近商户（Redis GEO）
  • 用户签到（Redis Bitmap）
  • 退出登录

个人心得：第一阶段的核心是理解 Token + Redis + 拦截器 三者协作的认证流程。相比传统 Session 方案，Token 方案天然解耦了服务端状态，配合 Redis TTL 能优雅地控制会话生命周期。ThreadLocal 管理用户上下文虽然便捷，但要记得在请求结束后及时 remove，避免内存泄漏，这在 Tomcat 线程池复用场景下尤其重要。
