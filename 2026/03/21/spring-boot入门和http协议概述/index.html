    • 模块一：SpringBootWeb 入门

      核心定义

      • 概念：Spring Boot Web 是基于 Spring Boot 框架的快速构建 Web 应用的入门实践，通过自动配置和起步依赖简化了传统 Spring MVC 的开发流程。

      关键知识点

      • 起步依赖：使用 spring-boot-starter-web 依赖快速初始化 Web 项目，自动集成 Tomcat 和 Spring MVC。

      • 自动配置：通过 @EnableAutoConfiguration 注解自动配置嵌入式服务器和 MVC 路由，无需手动配置 XML。

      • RESTful API 开发

        ：使用

        和

        注解定义接口，支持 JSON 数据序列化。

      • 配置文件：通过 application.properties 或 application.yml 配置端口、日志级别等，如 server.port=8080。

      • 启动方式：通过 mvn spring-boot:run 或直接运行主类（含 @SpringBootApplication 注解）启动应用。

      重要原理

      • 原理机制：Spring Boot 的自动配置基于条件注解（如 @ConditionalOnClass）检测类路径，动态加载配置类；起步依赖通过 BOM 管理版本，避免依赖冲突，实现“约定优于配置”的设计思想。

      模块二：HTTP 协议 - 请求协议

      核心定义

      • 概念：HTTP 请求协议定义了客户端向服务器发送请求的格式规范，包括方法、URL、头部和请求体，用于获取或操作资源。

      关键知识点

      • 请求方法：GET（获取资源）、POST（提交数据）、PUT（更新资源）、DELETE（删除资源）等，其中 GET 无请求体，POST/PUT 有体。

      • 请求头部：Content-Type（指定请求体格式，如 application/json）、User-Agent（标识客户端）、Authorization（携带认证令牌）。

      • URL 结构：包含协议（https）、主机（example.com）、路径（/api/users）和查询参数（?id=1）。

      • 请求体

        ：

        POST/PUT

        请求中携带的实体数据，常见格式为 JSON 或

        。

      • 状态码：请求处理结果的初始标识（如 200 成功、404 未找到），但实际状态由响应协议返回。

      重要原理

      • 原理机制：HTTP 是无状态协议，每个请求独立处理；请求方法定义操作类型（如 GET 仅读取），头部传递元数据（如内容类型），体携带数据，服务器根据方法和路径路由到对应处理逻辑。

      模块三：HTTP 协议 - 响应协议

      核心定义

      • 概念：HTTP 响应协议定义了服务器对客户端请求的响应格式规范，包括状态码、头部和响应体，用于返回处理结果和资源数据。

      关键知识点

      • 状态码：2xx（成功，如 200 OK）、4xx（客户端错误，如 404 Not Found）、5xx（服务器错误，如 500 Internal Server Error）。

      • 响应头部：Content-Type（指定响应体格式，如 application/json）、Cache-Control（控制缓存策略）、Set-Cookie（设置浏览器 Cookie）。

      • 响应体：包含实际返回数据，常见格式为 JSON、XML 或 HTML。

      • 常见状态码：201 Created（资源创建成功）、401 Unauthorized（认证失败）、302 Found（临时重定向）。

      重要原理

      • 原理机制：服务器根据请求内容生成响应，状态码指示处理结果；头部控制客户端行为（如缓存或安全策略），响应体携带数据，整个过程遵循 RFC 7230 规范，确保跨平台兼容性。

      模块四：SpringBootWeb 案例

      核心定义

      • 概念：通过一个完整的 Spring Boot Web 案例，演示从需求分析到代码实现的全流程，涵盖请求处理、响应返回及常见问题排查。

      关键知识点

      • 需求说明：实现一个用户管理接口，支持增删改查操作，如 /api/users/{id} 获取单个用户。

      • 代码实现

        ：

      • @ResponseBody：将方法返回值直接写入响应体，避免视图解析，常用于 REST API。

      • 问题分析：如 404 Not Found 可能因路径不匹配或控制器未扫描；500 Internal Server Error 可能因业务逻辑异常未捕获。

      重要原理

      • 原理机制：通过 @RestController 和 @RequestMapping 实现 URL 映射，@RequestBody 解析请求体，@ResponseBody 输出响应体，结合 ResponseEntity 精确控制状态码和头信息。

      模块五：分层解耦

      核心定义

      • 概念：通过分层架构（如 Controller、Service、DAO）实现代码解耦，提升可维护性和可测试性。

      关键知识点

      • 三层架构

        ：

        • Controller 层：接收请求，调用 Service。
        • Service 层：处理业务逻辑。
        • DAO 层：操作数据库。

      • 代码拆分

        ：

      • IOC & DI 入门：Spring 容器管理 Bean 生命周期，通过 @Autowired 实现依赖注入。

      • Bean 声明：使用 @Component, @Service, @Repository 注解声明组件。

      • 组件扫描：通过 @SpringBootApplication 启用包扫描，自动注册组件。

      • @Autowired 用法：字段、构造函数或方法上注入依赖，注意循环依赖问题。

      重要原理

      • 原理机制：IOC（控制反转）将对象创建权交给容器，DI（依赖注入）通过容器自动注入依赖，降低耦合度，提高代码复用性和可测试性。
